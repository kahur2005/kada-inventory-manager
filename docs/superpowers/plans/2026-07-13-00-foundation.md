# LogistiQ Plan 0: Foundation & Scaffolding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a working monorepo skeleton (Express API + React/Vite client) with all 8 Mongoose data models, DB connectivity, JWT auth middleware, and test harnesses in place, so Plans 1–4 (Auth/RBAC, Master Data & Stock, Boxes/QR, Driver/Tracking/Dashboard) can build features on top without re-deriving schema or plumbing.

**Architecture:** `/server` is an Express app talking to MongoDB Atlas via Mongoose, structured as `models/ middleware/ routes/ controllers/ config/`. `/client` is a Vite + React 18 app using `react-router-dom` v6 and a Context-based auth store. Both are independent npm packages under one repo root with a thin root `package.json` for running them together in dev.

**Tech Stack:** React 18 + Vite, react-router-dom v6, Express.js, Mongoose, MongoDB Atlas, `jsonwebtoken`, `bcryptjs`, `cors`, `dotenv`, `uuid`. Backend tests: Jest + Supertest + `mongodb-memory-server`. Frontend tests: Vitest + `@testing-library/react` + jsdom.

## Global Constraints

- Stack is fixed: React (Vite) + Express.js + MongoDB Atlas (Mongoose). Do not substitute (no Next.js, no Fastify, no Prisma).
- Auth: bcrypt password hashing + JWT, token stored in `localStorage` on the client, 4 assignable roles (`superadmin`, `warehouse_admin`, `store_admin`, `driver`) plus `unassigned`.
- Repo layout is a monorepo: `/client` (Vite React app) and `/server` (Express app with `routes/ controllers/ models/ middleware/ seed.js`).
- QR payloads must only ever contain `{ type: "box"|"driver", id, token }` — never item contents. This constrains the `Box.qrToken` / `User.driverQrToken` design now, even though QR generation itself lands in Plan 3.
- `WarehouseStock` and `StoreStock` each need a **unique compound index** on `(warehouse, item)` / `(store, item)` — enforced at the schema level in this plan.
- Deploy target for the demo is local + an ngrok HTTPS tunnel (camera access requires HTTPS) — not relevant to this plan's code, but do not hardcode `localhost` in a way that breaks tunneling (client API base URL must be env-driven).
- Every role-gated endpoint added in later plans will use `authRequired` + `requireRole(...)` middleware defined in this plan — their exact signatures are load-bearing for every later plan.

### Task 2: Mongo connection + Jest test harness

**Files:**
- Create: `server/config/db.js`
- Create: `server/tests/setup.js`
- Create: `server/jest.config.js`

**Interfaces:**
- Produces: `connectDB(uri): Promise<mongoose.Connection>` from `config/db.js`, used by `server.js` (Task 5) and never by tests directly.
- Produces: `server/tests/setup.js`, a module that (via `beforeAll`/`afterEach`/`afterAll`) spins up an in-memory MongoDB and points `mongoose` at it, and also defaults `process.env.JWT_SECRET` to `'test-secret'` at module-load time (before any test runs) so controller tests in Plans 1–4 that sign/verify JWTs never depend on a real `.env`. **Convention for every test file in this project (models, middleware, and all controller tests in Plans 1–4): put `require('../setup');` as the first line.** Jest applies hooks registered by a required module just as if they were declared inline in the test file, so this needs no special Jest config key — it works with the default `testEnvironment: 'node'` alone. Tests never touch the real Atlas cluster.

- [ ] **Step 1: Write `server/config/db.js`**

```js
const mongoose = require('mongoose');

async function connectDB(uri) {
  mongoose.set('strictQuery', true);
  await mongoose.connect(uri);
  return mongoose.connection;
}

module.exports = { connectDB };
```

- [ ] **Step 2: Write `server/tests/setup.js`**

```js
process.env.JWT_SECRET = process.env.JWT_SECRET || 'test-secret';

const mongoose = require('mongoose');
const { MongoMemoryServer } = require('mongodb-memory-server');

let mongod;

beforeAll(async () => {
  mongod = await MongoMemoryServer.create();
  await mongoose.connect(mongod.getUri());
});

afterEach(async () => {
  const collections = mongoose.connection.collections;
  for (const key in collections) {
    await collections[key].deleteMany({});
  }
});

afterAll(async () => {
  await mongoose.connection.dropDatabase();
  await mongoose.connection.close();
  await mongod.stop();
});
```

The longer `testTimeout` gives `mongodb-memory-server` room to download/boot the in-memory Mongo binary on first run.

- [ ] **Step 2: Implement `server/middleware/auth.js`**

```js
const jwt = require('jsonwebtoken');
const User = require('../models/User');

function signToken(user) {
  return jwt.sign({ id: user._id.toString(), role: user.role }, process.env.JWT_SECRET, {
    expiresIn: '7d',
  });
}

async function authRequired(req, res, next) {
  const header = req.headers.authorization || '';
  const [scheme, token] = header.split(' ');

  if (scheme !== 'Bearer' || !token) {
    return res.status(401).json({ message: 'Missing or malformed Authorization header' });
  }

  let decoded;
  try {
    decoded = jwt.verify(token, process.env.JWT_SECRET);
  } catch (err) {
    return res.status(401).json({ message: 'Invalid or expired token' });
  }

  const user = await User.findById(decoded.id).lean();
  if (!user) {
    return res.status(401).json({ message: 'User no longer exists' });
  }

  req.user = {
    id: user._id.toString(),
    role: user.role,
    warehouse: user.warehouse ? user.warehouse.toString() : null,
    store: user.store ? user.store.toString() : null,
    name: user.name,
    email: user.email,
  };
  next();
}

function requireRole(...roles) {
  return (req, res, next) => {
    if (!req.user || !roles.includes(req.user.role)) {
      return res.status(403).json({ message: 'Forbidden' });
    }
    next();
  };
}

module.exports = { authRequired, requireRole, signToken };
```

```jsx
import { createContext, useContext, useEffect, useState, useCallback } from 'react';
import apiClient from '../api/client';

const AuthContext = createContext(null);
const TOKEN_KEY = 'logistiq_token';

function decodeJwtPayload(token) {
  try {
    return JSON.parse(atob(token.split('.')[1]));
  } catch {
    return null;
  }
}

export function AuthProvider({ children }) {
  const [token, setToken] = useState(() => localStorage.getItem(TOKEN_KEY));
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    if (!token) {
      setUser(null);
      setLoading(false);
      return;
    }
    const payload = decodeJwtPayload(token);
    if (!payload) {
      setUser(null);
      setLoading(false);
      return;
    }
    // Plan 1 replaces this stub with `apiClient.get('/auth/me')`.
    setUser({ id: payload.id, role: payload.role });
    setLoading(false);
  }, [token]);

  const login = useCallback(async (email, password) => {
    const res = await apiClient.post('/auth/login', { email, password });
    localStorage.setItem(TOKEN_KEY, res.data.token);
    setToken(res.data.token);
    return res.data;
  }, []);

  const logout = useCallback(() => {
    localStorage.removeItem(TOKEN_KEY);
    setToken(null);
    setUser(null);
  }, []);

  return (
    <AuthContext.Provider value={{ user, token, loading, login, logout }}>
      {children}
    </AuthContext.Provider>
  );
}

export function useAuth() {
  const ctx = useContext(AuthContext);
  if (!ctx) throw new Error('useAuth must be used within AuthProvider');
  return ctx;
}
```
